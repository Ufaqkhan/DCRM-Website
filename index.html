Keep It Frozen: Domain-Routed Conditional Residual Modulation for Multi-Domain Vision Transformers

DCRM-ViT keeps a pre-trained ViT frozen and adapts it per sample through a domain router, a parameter synthesizer, and lightweight residual modulation blocks — aiming for robust performance across medical and natural imagery with fast, update-free inference.

Abstract

One frozen backbone for every domain

Medical imaging presents significant challenges due to acoustic shadows, motion blur, and indistinct boundaries between anatomical structures. Pre-trained Vision Transformers (ViTs) offer strong general-purpose feature representations, but adapting them to specialized domains — particularly medical imaging — typically requires extensive fine-tuning that risks degrading their original capabilities. We introduce DCRM-ViT (Domain-Routed Conditional Residual Modulation for Vision Transformers), a framework that keeps the ViT backbone entirely frozen while learning lightweight, input-conditioned residual corrections.

A domain router predicts soft domain membership weights from the input features. A parameter synthesizer network (PSN) converts these weights into low-rank residual parameters that are injected into residual modulation blocks (RMBs) attached to selected transformer layers. The entire system is trained end-to-end via bi-level optimization: an inner loop adapts task-specific parameters while an outer loop learns routing, synthesis, and initialization so that corrections generalize across domains.

DCRM-ViT achieves state-of-the-art results across fine-grained classification, zero-shot evaluation, cross-domain transfer, and medical image segmentation — all while adding only 3.3M trainable parameters and requiring no test-time updates. The method demonstrates that domain-aware, sample-level modulation of a frozen backbone is a powerful and efficient alternative to full fine-tuning.

The Problem

Why Can't One Model Handle All Domains?

Pre-trained Vision Transformers excel on natural images, but medical imaging demands entirely different visual understanding — acoustic shadows, speckle noise, motion blur, and subtle anatomical boundaries. Standard fine-tuning adapts to one domain but catastrophically forgets the other.

1

Domain Forgetting

Fine-tuning on medical images degrades accuracy on natural benchmarks (CIFAR, Caltech, etc.).

2

Heavy Compute

Full fine-tuning updates 86M+ parameters, requiring long training and large GPU memory.

Limitations

Why Existing PEFT Methods Fall Short

Parameter-efficient methods (LoRA, AdaptFormer, Prompt Tuning) reduce compute, but they apply static corrections — the same adaptation regardless of which domain the input belongs to. None of them route information dynamically based on the input sample itself.

❌ Static PEFT

Fixed low-rank updates shared across all inputs. Cannot distinguish chest X-rays from natural photos at the adaptation level.

✅ DCRM-ViT

Input-conditioned routing generates unique residual corrections per sample. Medical and natural inputs receive different adaptations automatically.

Our Solution

DCRM-ViT

Freeze the backbone. Route the adaptation.

Given any input image, a Domain Router first predicts which domain mixture it belongs to. A Parameter Synthesizer then generates sample-specific low-rank residuals. These are injected via Residual Modulation Blocks into the frozen ViT — producing domain-aware features without ever updating the backbone weights.

Architecture

Three Lightweight Modules — Zero Backbone Updates

1

Domain Router

Predicts soft domain weights from the input features, enabling mixture-of-experts-style routing without discrete branching.

2

Parameter Synthesizer

Converts domain weights into unique low-rank residual parameters for each sample, ensuring input-conditioned modulation.

3

Residual Modulation Blocks

Lightweight adapters that attach synthesized corrections to selected transformer projections inside the frozen backbone.

Deep Dive

Residual Modulation in Detail

Each RMB contains a TALer (Task-Aligned Low-rank) unit that produces a residual correction from synthesized parameters. This correction is added to the frozen transformer's output.

Optional domain-aware attention biasing lets the model attend differently based on domain context, while gating ensures gradual integration of corrections during training.

Training

Bi-Level Optimization for Cross-Domain Generalization

The model separates what to learn (task-level) from how to learn it (meta-level), enabling adaptation that generalizes across domains rather than overfitting to one.

Inner Loop

Adapts residual modulation parameters for the current task using standard gradient descent.

Outer Loop

Updates the domain router, synthesizer, initial residual states, and learning-rate meta-parameters so corrections transfer across domains.

Clinical Challenges

What makes medical adaptation hard

The paper targets a hard setting: adapting a single vision model across both medical and natural domains without sacrificing the general visual competence of the frozen backbone.

1

One frozen backbone across domains

A single transformer backbone is used for both medical and natural imagery, preserving broad visual competence while adapting to challenging medical inputs.

2

Per-sample domain-aware modulation

A domain router predicts soft domain weights, and a parameter synthesizer turns those weights into low-rank residual parameters for conditional adaptation.

3

Strong results with modest compute

Gains across fine-tuning, zero-shot evaluation, cross-domain transfer, and segmentation while keeping trainable parameters and training cost low.

Method

Frozen backbone, input-conditioned adaptation

DCRM-ViT adds small, domain-aware modules only where they are needed. The backbone remains frozen, while per-sample corrections are synthesized from the current input.

1

Frozen ViT backbone

The pre-trained encoder remains frozen so the model retains general visual features and keeps inference behavior stable.

2

Domain Router (DR)

A lightweight router predicts soft domain membership from the current features rather than relying on hard task-specific branching.

3

Parameter Synthesizer (PSN)

The PSN converts domain information into sample-specific low-rank parameters for the residual modulation blocks.

4

Residual Modulation Blocks

RMBs attach task-aligned low-rank corrections to selected transformer projections, with optional domain-aware attention biasing.

5

Bi-level optimization

An inner loop adapts task parameters, while an outer loop optimizes domain-level routing and initialization so corrections generalize.
